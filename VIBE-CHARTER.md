## 核心感受（North Star）

> 「像有个懂我的朋友在旁边」

用户每次打开，不是在用工具，是在和一个真正理解他的人说话。
这个朋友安静、专注、不打扰，但在你需要的时候总能接住你的想法。

**那个打动人的瞬间：**
通勤地铁上，一个想法突然闪过，三秒钟语音录下来，
朋友（AI）立刻轻声回应：「这个方向有意思，你有没有想过……」
那一刻感觉思维被接住了，不会消失了。

## 关键词定义（Glossary）

| 词 | 在本项目中的含义 | 反例（禁止出现） |
|---|---|---|
| **专注** | 界面只呈现当下最重要的事，没有噪音 | 仪表盘堆满数字和图表 |
| **丝滑** | 操作流程零摩擦，想到就能做到，动画自然不刻意 | 多步骤确认弹窗、加载等待 |
| **温暖** | 字体、配色、措辞像一个真实的人在说话 | 冰冷的系统提示、英文错误代码 |

## 三条红线（不可妥协）

1. **数据不出本地** — 用户内容永远存在用户自己的设备/账户里，不经过任何第三方服务器（AI API 调用除外，且用户知情）
2. **没有广告和推荐算法** — 界面里永远不出现任何推广内容，AI 的建议只服务于用户本人的目标
3. **界面克制，不花哨** — 每一个 UI 元素必须有存在的理由，装饰性元素严格控制

## 典型使用场景（Persona Story）

**用户：** 35岁，在城市通勤，脑子里有很多想法但容易忘
**设备：** 主力手机，偶尔 iPad 和桌面
**场景：** 地铁上站着，一只手抓扶手，另一只手拿手机
**心情：** 刚开完会，脑子还在转，有个想法想抓住
**动作：** 单手点开 App → 语音说出想法 → AI 轻声回应 → 放进口袋
**感受：** 那个想法被接住了，不会消失了，今天的脑子清空了一点

## 技术红线

**必须采用：**
- React + Vite（已验证）
- 本地优先存储（localStorage → 未来可选 IndexedDB）
- Web Speech API（语音输入，无需第三方）
- 纯前端部署（Vercel / Netlify，无后端）

**明确禁止：**
- 任何用户行为数据上报
- 第三方广告 SDK
- 强制注册/登录墙（初期）
- 超过 3 个直接依赖的 UI 组件库

---

## 验收标准（Vibe Check）

一个功能「vibe 对了」的判断依据：
1. **单手可完成** — 手机竖屏，拇指不需要跨越屏幕
2. **3 秒内完成核心操作** — 从打开到录入完成
3. **AI 回应像朋友不像机器** — 用口语，不用条列式报告
4. **看着舒服** — 给不认识这个项目的人看，他说「这个好看」
